📋 Deskripsi Class Diagram

Class Diagram ini menggambarkan struktur kelas dan hubungan antar kelas dalam sistem rental mobil, meliputi:

👤 User

Mengelola data pengguna sistem (Admin dan Manager). Memiliki fungsi login, logout, dan manajemen password.

⭐ Merk

Mengelola data merk mobil. Setiap mobil harus memiliki satu merk.

🚗 Mobil

Mengelola data mobil rental dengan atribut lengkap seperti plat nomor, tarif, status, dan foto. Berelasi dengan Merk.

👨‍✈️ Supir

Mengelola data supir dengan informasi pribadi dan tarif. Bersifat opsional dalam transaksi (lepas kunci).

📝 Transaksi

Mengelola transaksi rental mobil. Berelasi dengan Mobil (wajib) dan Supir (opsional). Menghitung total biaya dan durasi sewa.

👔 Manager

Role khusus untuk melihat laporan, statistik, dan monitoring. Extends dari User dengan akses read-only.

🔗 Relasi Antar Kelas

  • Mobil - Merk: Relasi 1:1 (Setiap mobil memiliki satu merk)
  • Transaksi - Mobil: Relasi 1:1 (Setiap transaksi menyewa satu mobil)
  • Transaksi - Supir: Relasi 0..1:0..1 (Transaksi bisa dengan atau tanpa supir)
  • Manager - User: Inheritance (Manager extends User)

📌 Catatan Penting

  • Status Mobil: bebas (tersedia) atau jalan (sedang disewa)
  • Status Transaksi: mulai (aktif) atau selesai (completed)
  • Lepas Kunci: idsupir = 0 untuk rental tanpa supir
  • Role User: Admin (full access) dan Manager (read-only)
