ℹ️ Tentang Activity Diagram:
Activity Diagram adalah diagram yang menggambarkan alur kerja (workflow) atau aktivitas dari sebuah sistem, proses bisnis, atau use case. Diagram ini menunjukkan urutan aktivitas dan keputusan yang terjadi dalam proses.

📝 Diagram 1: Tambah Transaksi Rental Mobil

Activity diagram ini menggambarkan alur lengkap proses penambahan transaksi rental mobil baru, mulai dari admin login hingga transaksi berhasil disimpan.

Alur Utama:
Login → Buka Form → Isi Data Pelanggan → Pilih Tanggal → Pilih Mobil & Supir → Sistem Hitung Biaya → Simpan → Ubah Status Mobil → Selesai

✅ Validasi Tanggal

Sistem memeriksa apakah tanggal kembali lebih awal dari tanggal sewa. Jika tidak valid, proses dihentikan.

✅ Validasi Data

Sistem memeriksa kelengkapan data sebelum menyimpan. Semua field wajib harus diisi.

💰 Perhitungan Otomatis

Sistem otomatis menghitung jumlah hari dan total biaya berdasarkan tarif mobil dan supir.

🚗 Update Status

Status mobil otomatis berubah menjadi "sedang disewa" setelah transaksi disimpan.

✅ Diagram 2: Selesaikan Transaksi Rental

Activity diagram ini menggambarkan proses penyelesaian transaksi rental, termasuk konfirmasi admin dan pengembalian status mobil.

Alur Utama:
Login → Lihat Daftar → Pilih Transaksi Aktif → Klik Selesai → Konfirmasi → Ubah Status Transaksi → Ubah Status Mobil → Selesai

⚠️ Konfirmasi

Admin harus mengkonfirmasi sebelum transaksi diselesaikan untuk menghindari kesalahan.

🔄 Update Otomatis

Status transaksi dan mobil diperbarui secara otomatis dalam satu proses.

🚗 Mobil Tersedia

Mobil langsung tersedia untuk disewa kembali setelah transaksi selesai.

🔒 Tombol Hilang

Tombol "Selesai" otomatis hilang setelah transaksi diselesaikan.

✏️ Diagram 3: Edit Transaksi Rental

Activity diagram ini menggambarkan proses pengubahan data transaksi yang sudah ada, dengan validasi status transaksi.

Alur Utama:
Login → Lihat Daftar → Klik Edit → Cek Status → Ubah Data → Hitung Ulang → Simpan → Selesai

🔒 Validasi Status

Transaksi yang sudah selesai tidak dapat diedit untuk menjaga integritas data.

🔄 Perhitungan Ulang

Sistem otomatis menghitung ulang biaya jika tanggal atau mobil/supir diubah.

❌ Opsi Batal

Admin dapat membatalkan perubahan kapan saja sebelum menyimpan.

✅ Validasi Data

Data yang diubah tetap divalidasi sebelum disimpan ke database.

🗑️ Diagram 4: Hapus Transaksi Rental

Activity diagram ini menggambarkan proses penghapusan transaksi, termasuk pengembalian status mobil jika transaksi masih aktif.

Alur Utama:
Login → Lihat Daftar → Klik Hapus → Konfirmasi → Cek Status → Kembalikan Status Mobil (jika aktif) → Hapus Data → Selesai

⚠️ Konfirmasi Hapus

Admin harus mengkonfirmasi penghapusan untuk menghindari penghapusan tidak sengaja.

🔍 Cek Status

Sistem memeriksa status transaksi sebelum menghapus untuk menentukan aksi selanjutnya.

🚗 Kembalikan Mobil

Jika transaksi aktif, status mobil dikembalikan menjadi "tersedia" sebelum dihapus.

🗑️ Hapus Permanen

Data transaksi dihapus permanen dari database dan tidak dapat dikembalikan.
