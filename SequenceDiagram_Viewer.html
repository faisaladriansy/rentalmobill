Proses Transaksi Rental Mobil - Alur Interaksi Sistem

📝 Diagram 1: Proses Tambah Transaksi

Sequence diagram ini menggambarkan alur proses ketika admin menambahkan transaksi rental mobil baru, mulai dari membuka form, memilih mobil dan supir, hingga menyimpan data ke database.

Alur Proses:

1

Inisialisasi Form

Admin membuka form tambah transaksi. Sistem mengambil data mobil yang tersedia (status 'bebas') dan daftar supir dari database.

2

Input Data Pelanggan

Admin mengisi data pelanggan (nama, KTP, HP, alamat), memilih tanggal sewa & kembali, memilih mobil, dan memilih supir (opsional untuk lepas kunci).

3

Validasi Data

Controller memvalidasi input data. Jika tidak valid, tampilkan error. Jika valid, lanjut ke proses perhitungan.

4

Hitung Total Biaya

Sistem menghitung selisih hari dan total biaya: (tarif mobil + tarif supir) × jumlah hari

5

Simpan Transaksi

Data transaksi disimpan ke database dengan status 'mulai' (transaksi aktif).

6

Update Status Mobil

Status mobil diubah menjadi 'jalan' (sedang disewa) agar tidak bisa disewa oleh pelanggan lain.

✅ Diagram 2: Proses Selesaikan Transaksi

Sequence diagram ini menggambarkan alur proses ketika admin menyelesaikan transaksi rental mobil, termasuk update status transaksi dan mengembalikan status mobil menjadi tersedia.

Alur Proses:

1

Lihat Daftar Transaksi

Admin membuka halaman transaksi. Sistem menampilkan semua transaksi dengan JOIN ke tabel mobil dan supir.

2

Pilih Transaksi Aktif

Admin mengklik tombol "Selesai" pada transaksi yang statusnya 'mulai'. Sistem menampilkan konfirmasi.

3

Ambil Data Transaksi

Controller mengambil detail transaksi berdasarkan ID untuk mendapatkan informasi mobil yang disewa.

4

Update Status Transaksi

Status transaksi diubah menjadi 'selesai' di database.

5

Kembalikan Status Mobil

Status mobil dikembalikan menjadi 'bebas' (tersedia) sehingga bisa disewa kembali oleh pelanggan lain.

🎭 Aktor

  • Admin: Pengguna yang mengelola transaksi
  • Memiliki akses penuh untuk CRUD transaksi
  • Dapat melihat dan mengelola semua data

🔧 Komponen Sistem

  • View: Tampilan antarmuka pengguna
  • Controller: Logika bisnis dan validasi
  • Model: Akses dan manipulasi data
  • Database: Penyimpanan data

📊 Status Penting

  • Status Mobil: 'bebas' atau 'jalan'
  • Status Transaksi: 'mulai' atau 'selesai'
  • Status otomatis berubah saat transaksi dibuat/diselesaikan

🚗 Diagram 3: Pengelolaan Data Mobil dan Merk

Sequence diagram ini menggambarkan alur proses pengelolaan data mobil dan merk, termasuk operasi tambah, ubah, dan hapus data.

Alur Proses Data Merk:

1

Melihat Data Merk

Admin mengakses halaman Data Merk. Sistem menampilkan semua merk dalam bentuk tabel.

2

Kelola Data Merk

Admin dapat menambah merk baru, mengubah nama merk, atau menghapus merk yang tidak digunakan.

Alur Proses Data Mobil:

1

Melihat Data Mobil

Admin mengakses halaman Data Mobil. Sistem menampilkan kartu mobil dengan foto dan informasi lengkap.

2

Tambah/Edit Mobil

Admin mengisi data mobil (plat, nama, merk, warna, tahun, status, foto). Sistem menyimpan ke database.

👨‍✈️ Diagram 4: Pengelolaan Data Supir

Sequence diagram ini menggambarkan alur proses pengelolaan data supir, termasuk operasi tambah, ubah, dan hapus data supir dengan validasi penggunaan dalam transaksi.

Alur Proses:

1

Melihat Data Supir

Admin mengakses halaman Data Supir. Sistem menampilkan tabel berisi nama, alamat, kontak, dan foto supir.

2

Menambah Supir Baru

Admin mengisi form data supir (nama, alamat, kontak, foto) dan menyimpan ke database.

3

Mengubah Data Supir

Admin memilih supir yang akan diubah, sistem menampilkan form terisi data lama, admin mengubah dan menyimpan.

4

Menghapus Data Supir

Admin menghapus supir. Sistem memeriksa apakah supir sedang digunakan dalam transaksi aktif. Jika ya, penghapusan ditolak.
