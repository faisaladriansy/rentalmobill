📋 Deskripsi Class Diagram

Class Diagram ini menggambarkan struktur lengkap sistem rental mobil berdasarkan analisis model dan database:

👤 User

Mengelola data pengguna sistem untuk autentikasi login.

  • iduser (PK)
  • username, password
  • lastlogin, stuser

⭐ Merk

Mengelola data merk mobil (Toyota, Honda, Daihatsu, dll).

  • idmerk (PK)
  • namamerk
  • namamerk_seo

🚗 Mobil

Mengelola data mobil rental dengan status ketersediaan.

  • idmobil (PK), idmerk (FK)
  • type, tahunproduksi
  • platnomer, kursi
  • tarif, lembur
  • stmobil: bebas/jalan

👨‍✈️ Supir

Mengelola data supir dengan tarif harian dan lembur.

  • idsupir (PK)
  • namasupir, tgllahir
  • alamat, noktp
  • tarif, lembur

📝 Transaksi

Mengelola transaksi rental mobil dengan perhitungan otomatis.

  • idtransaksi (PK)
  • idmobil (FK), idsupir (FK)
  • namapelanggan, noktp, nohp
  • tglsewa, tglkembali, selisih
  • total
  • sttransaksi: mulai/selesai

📊 Manager

Kelas khusus untuk dashboard, laporan, dan statistik.

  • getStatistik()
  • getMobilTersedia()
  • getMobilJalan()
  • getLaporanTransaksi()
  • getGrafikBulanan()
  • getMobilPopuler()

🔗 Relasi Antar Kelas

  • Merk → Mobil: Relasi 1:* (Satu merk dapat memiliki banyak mobil)
  • Mobil → Transaksi: Relasi 1:* (Satu mobil dapat disewa berkali-kali)
  • Supir → Transaksi: Relasi 0..1:* (Transaksi opsional menggunakan supir)

📌 Catatan Penting

  • Status Mobil: bebas (tersedia) atau jalan (sedang disewa)
  • Status Transaksi: mulai (aktif) atau selesai (sudah dikembalikan)
  • Lepas Kunci: idsupir = 0 atau kosong untuk transaksi tanpa supir
  • Perhitungan Total: (tarif mobil × selisih hari) + (tarif supir × selisih hari)
